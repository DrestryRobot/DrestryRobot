机械 三维建模
===============

基本概念
-----------
三维建模需要根据机器人的功能需求设计相应的三维模型，采用自上而下的设计思路，先确定机器人整体设计方案，划分机器人各功能机构，再对各功能机构进行方案设计，依据详细的设计方案对各零部件进行三维建模，最后进行各机构的装配及机器人整体组装。

技术要点
-----------
对于机器人本体结构的三维建模，在设计之初，必须明确的是机器人的功能定位，这为之后的所有设计奠定了基础，比如，扫地机器人的功能定位在于扫地，那么在结构设计上我们就能清楚，其大体应该由可移动底盘和清洁机构两个部分组成，再比如，人形机器人是仿照人的行为动作来设计的，故其在结构上应该具有头、身体、双臂、双腿足等部分。由此，我们很明显可以看出来，对于机器人本地结构的三维建模，一旦我们明确了其功能定位，那么根据功能定位我们就能确定其的大体结构设计组成。（功能定位）

另外，确定了机器人的功能定位，划分了结构组成，也就明确了大致的设计方案，比如，扫地这个动作，比较好的设计方案是采用电动，驱动清洁刷头进行作业，再比如，抓取杯子这个动作，使用气动的话速度快力量大，使用电动的话精度高可控制，在此基础上，再结合我们功能定位的具体场景需求，比如是对时间要求高还是对控制精度有要求，也可以明确具体的设计方案。（设计方案）

可是，有时候，即使我们已经明确了功能定位和设计方案，依然不知道如何开始下手展开设计，此时，需要有一个大框架的概念，设计之初，不可能一眼就看到底，每一处的设计细节都应该是打磨之后形成的，所以在设计之初，机器人整体是什么样子的，应该具有的是一个整体的模糊的框架概念，比如大体是啥样子的形状，是规则形状的，还是不规则形状的，是比较大的，还是比较小的，是细长的，还是矮胖的，我们当然并不清楚具体的结构是什么样的，可是我们可以像捏橡皮泥一样，先有一个大概的雏形，然后对于每一处的细节进行打磨，最后整体协调，完成设计。（框架概念）

为了完成三维建模，我们有许多的建模工具可以选择，但是正如编程语言一样，各种编程语言也好，又或者是各种建模工具也罢，都只不过是一种实现目的的方法和手段，终点是一样的，路径各有不同而已。因此，当我们站在这样的角度去理解建模软件的时候，我们时刻就把思考的重心放在了三维模型本身上，我们应该考虑的是模型怎么设计更好，而不是这个设计我们能不能用建模工具实现。（建模工具）
